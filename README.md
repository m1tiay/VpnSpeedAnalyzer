# VPN Speed Analyzer

[English](README.en.md) | Русский

`VPN Speed Analyzer` — настольное приложение на `WPF + .NET 8` для контроля
качества VPN-подключения и выбора лучшего хоста по реальным сетевым замерам.

## О проекте кратко

- Делает периодические замеры через `speedtest.exe` (Ookla CLI) и `ipwho.is`/`ipapi.co`.
- Считает по каждому хосту единую оценку качества `D.Q.S (0..100)` под текущий сценарий использования.
- Показывает живой мониторинг (графики, таблица, рейтинг топ-3) и отдельную аналитику по всем хостам.
- Помогает быстро понять, какой VPN-узел выбрать прямо сейчас.

## Возможности

- **Мониторинг в реальном времени:** ping, jitter, loss, download, upload.
- **Графики:** компактные графики ping и jitter с историей.
- **Таблица результатов:** по каждому замеру — IP, страна, метрики, `D.Q.S`.
- **Рейтинг хостов:** топ-3 на вкладке `Мониторинг` + полный аналитический рейтинг на вкладке `Аналитика`.
- **Расшифровка оценки:** блок «Почему этот D.Q.S?» поясняет, как получился балл для выбранной строки.
- **Профили оценки:** `Универсальный`, `Игры`, `Стрим` — меняют веса метрик «на лету».
- **Определение IP/страны/ASN:** основной источник `ipwho.is`, резервный `ipapi.co`.
- **Надёжные замеры:** ретраи `speedtest`, статусы в UI, понятные ошибки.
- **Экспорт CSV:** общий список и топ-хостов.
- **Логи:** в `%LOCALAPPDATA%\VpnSpeedAnalyzer\logs` с ротацией и очисткой.

## Метрика D.Q.S

`D.Q.S = Daver Quality Score` — авторская сводная оценка качества канала по шкале `0..100`.

Каждая метрика сначала нормируется в шкалу `0..100`:

- ping, jitter, loss — *чем меньше, тем лучше*: `≤ ideal` → 100, `≥ worst` → 0, между ними — линейно.
- download, upload — *чем больше, тем лучше*: `≥ ideal` → 100, `≤ worst` → 0, между ними — линейно.

Текущие пороги:

| Метрика     | ideal | worst | Что измеряет           | Тренд            |
| ----------- | ----- | ----- | ---------------------- | ---------------- |
| ping (мс)   | 20    | 150   | задержка               | меньше — лучше   |
| jitter (мс) | 2     | 40    | стабильность задержки  | меньше — лучше   |
| loss (%)    | 0     | 5     | потери пакетов         | меньше — лучше   |
| download    | 400   | 20    | скорость загрузки      | больше — лучше   |
| upload      | 150   | 10    | скорость отдачи        | больше — лучше   |

Итоговый балл — **взвешенная сумма** нормализованных метрик. Веса задаются текущим профилем (см. ниже).
То есть формула простая:

```
D.Q.S = ping_score   * w_ping
      + jitter_score * w_jitter
      + loss_score   * w_loss
      + dl_score     * w_download
      + ul_score     * w_upload
```

Сумма весов в каждом профиле равна `1.00`, поэтому итог всегда в диапазоне `0..100`.

## Профили оценки

Профиль меняет приоритеты — то есть веса метрик в формуле `D.Q.S`.
Это позволяет одной и той же таблице замеров показать разный «лучший» хост для разных задач.

| Профиль        | Назначение                                     | ping | jitter | loss | download | upload |
| -------------- | ---------------------------------------------- | ---- | ------ | ---- | -------- | ------ |
| Универсальный  | Сбалансированный — общий интернет и VPN        | 0.30 | 0.20   | 0.25 | 0.15     | 0.10   |
| Игры           | Минимум задержки и потерь, скорости вторичны   | 0.42 | 0.28   | 0.20 | 0.06     | 0.04   |
| Стрим          | Стабильность канала и высокие скорости         | 0.20 | 0.10   | 0.15 | 0.35     | 0.20   |

Как читать строки:

- **Игры:** ping/jitter/loss дают `0.90` веса — даже отличная скорость не вытащит хост, если задержка плохая.
- **Стрим:** download/upload дают `0.55` веса — приоритет у скоростных хостов с приемлемой задержкой.
- **Универсальный:** сбалансированный — годится по умолчанию.

Профиль переключается в шапке приложения, рейтинг и рекомендация пересчитываются мгновенно.

## Требования

Для корректной работы необходимо подготовить окружение:

- **Утилита `speedtest.exe` (Ookla Speedtest CLI).**
  Приложение запускает её в фоне для замеров скорости и качества канала.
  Скачать с официального сайта: <https://www.speedtest.net/apps/cli>.
  Достаточно положить `speedtest.exe` рядом с `.exe` приложения —
  именно там она ищется в первую очередь. Прописывать утилиту в `PATH` необязательно
  (это используется только как резервный вариант, если файла нет рядом).
- **Запуск через тот же VPN-контур, который тестируете.**
  И само приложение, и `speedtest.exe` должны идти через VPN —
  иначе внешний IP, страна, ASN и метрики будут отражать реальный канал, а не VPN.
  Если трафик утилит обходит VPN, смена сервера/маршрута может не определяться.

## Быстрый старт

1. Клонировать репозиторий.
2. Собрать и запустить:
   - `dotnet build -c Release`
   - `dotnet run`
3. Положить `speedtest.exe` рядом с собранным `.exe` (или в `PATH`).
4. Нажать `Старт` в интерфейсе.
5. Переключать профиль и сравнивать хосты по `D.Q.S` и метрикам.

## Сборка релиза (Single EXE)

В GitHub Actions выполняется публикация standalone EXE для Windows (`win-x64`) с загрузкой артефакта.

Локальная команда публикации:

```
dotnet publish VpnSpeedAnalyzer.csproj -c Release -r win-x64 --self-contained true -o out
```

## Логи и приватность

- Логи лежат в `%LOCALAPPDATA%\VpnSpeedAnalyzer\logs` (ротация и автоочистка).
- Замеры делает локально установленная `speedtest.exe` — внешние сервисы используются только для определения IP/страны/ASN.
