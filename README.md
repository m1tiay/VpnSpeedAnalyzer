# VPN Speed Analyzer

[English](README.en.md) | Русский

`VPN Speed Analyzer` — настольное приложение на `WPF + .NET 8` для выбора лучшего VPN-хоста по реальным сетевым замерам.

## Коротко

Приложение в реальном времени сравнивает хосты по ping, jitter, packet loss, download/upload и считает `D.Q.S (0..100)` с учетом выбранного профиля.

## Требования

Для корректной работы необходимо подготовить окружение:

- **Утилита `speedtest.exe` (Ookla Speedtest CLI).**
  Приложение запускает её в фоне для замеров скорости и качества канала.
  Скачать с официального сайта: <https://www.speedtest.net/apps/cli>.
  Достаточно положить `speedtest.exe` рядом с `.exe` приложения —
  именно там она ищется в первую очередь. Прописывать утилиту в `PATH` необязательно
  (это используется только как резервный вариант, если файла нет рядом).
- **Запуск через тот же VPN-контур, который тестируете.**
  И само приложение, и `speedtest.exe` должны идти через VPN —
  иначе внешний IP, страна, ASN и метрики будут отражать реальный канал, а не VPN.
  Если трафик утилит обходит VPN, смена сервера/маршрута может не определяться.

## Возможности

- **Определение IP:** основной источник `ipwho.is`, резервный `ipapi.co`.
- **Надежные замеры:** ретраи speedtest, логирование причин ошибок, статусы в UI.
- **Оценка качества:** `D.Q.S` для каждого хоста.
- **Расшифровка:** `D.Q.S = Daver Quality Score` (авторская логика оценки качества канала).
- **Профили оценки:** `Универсальный`, `Игры`, `Стрим`.
- **Рейтинг хостов:** отдельная вкладка с топом и рекомендацией лучшего хоста.
- **Пояснение оценки:** блок «Почему этот D.Q.S?» для выбранной строки.
- **Визуализация:** графики ping и jitter.
- **Экспорт:** CSV для общего списка и CSV для топа хостов.
- **Логи:** хранение в `%LOCALAPPDATA%\VpnSpeedAnalyzer\logs` с ротацией и очисткой.

## Зачем это нужно

Выбор VPN-узла только по стране или названию провайдера часто дает нестабильную задержку и скорость.
`VPN Speed Analyzer` превращает сырые метрики в понятный рейтинг и практическую рекомендацию.

## Технологии

- **Платформа:** `WPF`, `.NET 8`
- **Графики:** `ScottPlot`
- **Архитектура:** MVVM-подход (`ViewModels`, `Models`, `Logic`, `Services`)
- **CI/CD:** GitHub Actions
- **Дистрибуция:** standalone single EXE

## Быстрый старт

1. Клонировать репозиторий.
2. Собрать и запустить:
   - `dotnet build -c Release`
   - `dotnet run`
3. Нажать `Старт` в интерфейсе.
4. Переключать профиль оценки и выбирать лучший хост из рейтинга.

## Сборка релиза (Single EXE)

В GitHub Actions выполняется публикация standalone EXE для Windows (`win-x64`) с загрузкой артефакта.

Локальная команда публикации:

`dotnet publish VpnSpeedAnalyzer.csproj -c Release -r win-x64 --self-contained true -o out`
